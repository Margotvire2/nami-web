# Creation Process — Créer un article SEO/GEO de A à Z

## Étape 1 — Brief éditorial

Avant d'écrire un seul mot, produire ce brief complet :

```
## Brief — [TITRE PROVISOIRE]

Mot-clé principal       : 
Volume estimé           : [utiliser Ubersuggest, Semrush ou Google Keyword Planner]
Difficulté SEO          : [faible / moyen / élevé]
Intention de recherche  : [informationnelle / navigationnelle / transactionnelle]
Type d'article          : [pilier / satellite / GEO]
Longueur cible          : [X mots]
Audience                : [patient / aidant / professionnel / mixte préciser]

Mots-clés secondaires :
1. [longue traîne 1]
2. [longue traîne 2]
3. [variante synonyme]
4. [variante patient/langage courant]
5. [variante GEO si pertinent]

Angle différenciant     : [Qu'est-ce qui rend CET article meilleur que les 3 premiers résultats Google ?]

Sources principales :
1. [Source rang 1 avec lien]
2. [Source rang 1 avec lien]
3. [Source rang 1 avec lien]

Concurrents top 3 (premiers résultats Google.fr) :
1. [URL + forces + faiblesses]
2. [URL + forces + faiblesses]
3. [URL + forces + faiblesses]

Articles internes à mailler :
1. /blog/[slug-existant-1]
2. /blog/[slug-existant-2]

CTA de fin              : [lien vers page service / contact / fiche parcours]
Visuels à produire      : [infographie / schéma / photo / illustration]
```

### Comment trouver le bon mot-clé

**Méthode rapide (sans outil payant)** :
1. Taper le sujet dans Google.fr
2. Noter les suggestions d'autocomplétion
3. Regarder "People Also Ask" (questions)
4. Regarder "Recherches associées" en bas de page
5. Le mot-clé idéal = volume suffisant + difficulté acceptable + intention claire

**Méthode avec outils** :
- Google Search Console (mots-clés qui amènent déjà du trafic)
- Ubersuggest (gratuit limité)
- Semrush / Ahrefs (payant, le plus précis)
- AlsoAsked.com (questions PAA gratuites)
- AnswerThePublic (variantes de questions)

### Analyse concurrentielle express

Pour chaque concurrent top 3, noter :
- Longueur de l'article (Ctrl+A → word count)
- Nombre de H2
- Sources citées ou non
- Date de publication / mise à jour
- Score EEAT apparent (auteur identifié ? expertise démontrée ?)
- Ce qui manque (= ton angle différenciant)

## Étape 2 — Plan détaillé

Produire le plan complet avant de rédiger :

```
# [H1 — 50-65 chars, mot-clé en début]

## Introduction (100-150 mots)
→ Accroche : [empathique, liée au vécu]
→ Chiffre clé : [incidence/prévalence France, sourcée]
→ Promesse : [ce que le lecteur saura après avoir lu]
→ Transition : [vers la première section]

## [H2 — Définition / Qu'est-ce que c'est]
→ Mot-clé secondaire ciblé : [xxx]
→ Points à couvrir : [définition accessible, mécanisme simplifié, qui est concerné]
→ Source : [xxx]

## [H2 — Symptômes / Signes d'alerte]
→ Mot-clé secondaire : [xxx]
→ Points : [symptômes en langage patient, signaux de gravité]
→ INTERDIT : posologies, noms de médicaments avec doses

## [H2 — Prise en charge / Parcours de soins]
→ Mot-clé secondaire : [xxx]
→ Points : [étapes du parcours, professionnels impliqués, durée, coordination]
→ Lien interne ici : [vers article sur la coordination / page service Nami]

## [H2 — Vivre avec / Conseils quotidiens]
→ Points : [conseils pratiques validés, ressources, associations]

## [H2 — Quand consulter]
→ Points : [situations de consultation, situations d'urgence, par qui commencer]

## Questions fréquentes (3-5)
→ Q1 : [question PAA — info NOUVELLE]
→ Q2 : [question PAA — info NOUVELLE]
→ Q3 : [question PAA — info NOUVELLE]
→ Q4 : [question PAA — info NOUVELLE si nécessaire]

## Sources (minimum 3, rang 1)
```

### Règles du plan

- Chaque H2 doit pouvoir fonctionner comme un article autonome (featured snippet potentiel)
- L'ordre suit le parcours mental du lecteur : comprendre → reconnaître → agir → vivre avec → quand s'inquiéter
- Le plan doit couvrir TOUT ce que les 3 premiers résultats Google couvrent + AU MOINS un angle qu'ils ne couvrent pas

## Étape 3 — Rédaction

### Introduction — Les 100-150 premiers mots

Structure APCR (Accroche, Problème chiffré, Contenu, Résultat) :

```
[Accroche empathique — 1 phrase qui parle au vécu du lecteur]

[Problème chiffré — incidence ou prévalence en France avec source]

[Contenu — ce que cet article couvre concrètement, en 1-2 phrases]

[Résultat — ce que le lecteur saura/pourra faire après avoir lu]
```

**Exemple** :
```
Votre enfant perd du poids sans raison apparente, refuse de manger, 
ou vous inquiétez de son rapport à la nourriture ?

Les troubles des conduites alimentaires touchent environ 600 000 
personnes en France, dont une proportion croissante d'enfants et 
d'adolescents (FFAB, 2023).

Ce guide détaille les signes d'alerte, le parcours de soins 
pluridisciplinaire et les professionnels à consulter pour une prise 
en charge adaptée.

À la fin de cette lecture, vous saurez reconnaître les premiers 
signaux et engager les bonnes démarches.
```

### Corps — Règles par section

**Chaque paragraphe** :
- 3-5 phrases maximum
- 1 idée par paragraphe
- Phrase d'ouverture = l'essentiel (le lecteur qui scanne doit comprendre en lisant les premières phrases)

**Listes à puces** :
- Maximum 7 items (au-delà, regrouper)
- Chaque item commence par un mot-clé ou un verbe d'action
- Pas de listes de 1-2 items (rédiger en phrases)

**Données chiffrées** :
- Toujours entre parenthèses : (Source, Année)
- Arrondies pour la lisibilité : "environ 15%" plutôt que "14,7%"
- Contextualisées : "1 personne sur 7" plus parlant que "14,3%"

**Liens internes** :
- Distribués naturellement dans le texte (pas regroupés en fin d'article)
- Ancre = 3-6 mots descriptifs du contenu cible
- Minimum 1 lien dans les 300 premiers mots

### FAQ — Les 3-5 questions

**Comment trouver les bonnes questions** :
1. Taper le mot-clé principal dans Google.fr
2. Copier les 4-5 questions "People Also Ask"
3. Vérifier que chaque question apporte une info ABSENTE du corps de l'article
4. Si les PAA ne sont pas pertinentes → utiliser AlsoAsked.com ou AnswerThePublic

**Format de chaque réponse** :
- 40-60 mots
- Réponse directe dès la première phrase (position 0 potentielle)
- 1 donnée chiffrée ou 1 fait concret
- Pas de renvoi vers le corps ("comme vu plus haut")

### Conclusion / CTA

- Pas de section "Conclusion" avec ce titre (Google le dévalorise)
- Résumer en 2-3 phrases les actions concrètes pour le lecteur
- CTA naturel vers la page de service, la prise de contact, ou un article lié
- Disclaimer médical en italique

## Étape 4 — Optimisation finale

### Relecture EEAT
- [ ] L'auteur est identifié avec credentials
- [ ] Au moins 1 mention d'expérience clinique/terrain
- [ ] Toutes les affirmations cliniques sont sourcées (rang 1)
- [ ] Date de publication ET date de révision présentes

### Relecture SEO
- [ ] Mot-clé dans : H1, intro, 1 H2, conclusion — densité < 1%
- [ ] 5+ variantes du mot-clé utilisées naturellement
- [ ] Meta title optimisé (50-65 chars)
- [ ] Meta description optimisée (145-155 chars)
- [ ] Slug optimisé (3-5 mots)
- [ ] 4+ liens internes avec ancres descriptives
- [ ] Schema JSON-LD préparé

### Relecture qualité
- [ ] Aucune phrase template (cf. liste dans correction-checklist.md)
- [ ] Aucune posologie ni nom commercial de médicament
- [ ] FAQ avec infos nouvelles (pas de doublons avec le corps)
- [ ] Sources complètes avec titre + organisme + année + lien
- [ ] Longueur atteinte (1500+ pilier, 800+ satellite, 600+ GEO)
- [ ] Transitions présentes entre chaque section
